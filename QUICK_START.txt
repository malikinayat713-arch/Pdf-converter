================================================================================
                            FEATURE SUMMARY
================================================================================

USER FLOW:
    1. User logs in (existing)
    2. Selects "Convert" or "Search" tab (NEW)
    3. Uploads PDF
    4. For Search: Types query → Gets results with page numbers
    5. Sees all matches highlighted with exact pages

KEY FEATURES (MVP):
    ✓ Mode selector (Convert vs Search)
    ✓ PDF upload (drag & drop, click, max 500MB)
    ✓ Case-insensitive search
    ✓ Page number tracking (exact page for each match)
    ✓ Match count per page
    ✓ Context snippet (80 chars before/after)
    ✓ Copy results to clipboard
    ✓ Mobile responsive
    ✓ Full Urdu/English bilingual support
    ✓ Loading states and error handling

TECH STACK:
    Backend:  Node.js + Express + pdf-parse + pdfjs-dist
    Frontend: React (existing)
    Design:   Matches existing blue/purple gradient aesthetic

Setup:

    cd backend && npm start              # Starts on :5000
    curl -X POST http://localhost:5000/api/pdf/extract -F "pdf=@test.pdf"

Test Frontend Locally:

Build for Production:
    cd frontend && npm run build

================================================================================
                         ARCHITECTURE AT A GLANCE
================================================================================

BACKEND FLOW:
    PDF Upload → Extract text + cache (1hr TTL) → Return pdfId
    Search Query → Search cached text with regex → Return page numbers

FRONTEND FLOW:
    Upload → Call /api/pdf/extract → Get pdfId → Enable search
    Search → Call /api/pdf/search → Display results by page number

API ENDPOINTS (NEW):
    POST /api/pdf/extract ............ Extract & cache PDF text
    POST /api/pdf/search ............ Search cached PDF
    GET /api/pdf/:pdfId/info ....... Get cache info
